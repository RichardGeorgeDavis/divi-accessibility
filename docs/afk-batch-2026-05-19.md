## 2.1.9 Dropdown Helper Visibility

- Live report: `.da11y-dropdown-toggle` was visible on <https://lucidity.design/>.
- Observed live stack from page assets: WordPress `6.9.4`, Divi `4.27.6`, Divi Pixel `2.50.1`, and WP Rocket cached/minified frontend assets.
- Reproduced on desktop viewport `1600x1000`: two empty `.da11y-dropdown-toggle` buttons were visible for `Solutions` and `Services`.
- Mobile viewport `390x844` did not inject the dropdown helper buttons.
- Static scan found the plugin's intentionally inserted helper UI in skip links, search labels/buttons, contact labels, slider controls, mobile-menu roles, and dropdown toggles; the dropdown toggle was the only visible extra helper artifact on the sampled live pages.
- Sampled live pages: `/`, `/pricing/`, `/contact/`, `/work/`, and `/blog/`. Each desktop page showed only the same two dropdown helpers; contact form labels remained clipped to screen-reader-only dimensions.
- Fix: make `.da11y-dropdown-toggle` visually hidden and pointer-inert by default, then visible and pointer-active on keyboard focus.
- Verification: injected the patched CSS into the live page. The helper reported default `opacity: 0` and `pointer-events: none`; after focus it reported `opacity: 1` and `pointer-events: auto`; pressing Enter still opened the submenu with `aria-expanded="true"`, `.et-hover`, and `.da11y-submenu-show`.
